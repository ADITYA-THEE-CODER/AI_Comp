Privacy & Data

Privacy Policy

CyberShield AI is built with a privacy-first architecture. We believe cybersecurity tools should not compromise user privacy while protecting users from threats.

Data Processing: All scans are processed temporarily in memory. No scan input or results are permanently stored on our servers by default.

LLM Interactions: When multiple LLMs analyze your input, the data is sent directly to the respective API endpoints under their own privacy policies. We do not store intermediary LLM responses.

Cookies & Tracking: We use minimal, essential cookies only for session management and theme preferences. No third-party tracking or analytics cookies are used.

Data Retention: Scan results are available during your session and are purged when the session ends. Users may optionally export results before closing the session.

Data Collection

What We Collect:

  • Scan input you provide (text, URLs, file content) — processed temporarily only
  • Context information you optionally provide — processed temporarily only
  • Aggregate anonymized statistics (threat categories, scan counts) — no personal data

What We Do NOT Collect:

  • Personal identification information
  • IP addresses linked to scan results
  • Browsing history or behavior patterns
  • Permanent storage of scan inputs or outputs

User Consent

By using CyberShield AI, you consent to the temporary processing of your input data for threat analysis purposes.

Temporary Processing

All data processing is ephemeral. When you submit a scan:

  1. Your input is encrypted in transit (TLS/SSL)
  2. It is held in secure memory for the duration of the analysis
  3. It is sent to configured LLM APIs over secure connections
  4. Results are assembled and delivered to you
  5. All temporary data is purged from memory after delivery
Zero Retention

By default, no data is written to disk. All processing occurs in volatile memory only.

Documentation

1. Starting a Scan

Navigate to the Scanners page from the top navigation. Choose the scanner module that matches your input type (Email, URL, File, etc.). Enter or paste the content you want analyzed in the input area.

2. Providing Context

Before each scan, you'll see a Context Box. This is where you can share additional information about the item being scanned — where you received it, whether it was expected, who sent it, etc. This helps our AI produce more accurate and relevant assessments.

3. Understanding Results

After analysis, you'll see: individual LLM responses, an aggregated verdict, a confidence score, and a human-friendly AI advisor opinion. The verdict ranges from Safe (green) to Critical (red) with varying confidence levels.

4. Multi-LLM Analysis

Each scan is processed by multiple AI models simultaneously. The system compares their outputs, resolves disagreements, and produces one unified, explainable verdict. This cross-verification significantly improves accuracy.

5. Dashboard & History

The Dashboard provides an overview of all your scans, threat statistics, and security tips. Your recent scan history is displayed for quick reference.

Is my data stored permanently?

No. All data is processed temporarily in memory and purged after results are delivered. See our Privacy page for full details.

How accurate are the scans?

Our multi-LLM approach typically achieves 85-95% accuracy depending on the threat type. Each result includes a confidence score so you can gauge reliability.

Which LLMs are used?

Currently: GPT-4o, Claude 3.5, and Gemini Pro. Additional models can be added to the pipeline via configuration.

Can I use this for my organization?

Yes. CyberShield AI is designed for both individual and organizational use. Contact us for enterprise deployment options.

What is n8n integration?

n8n is a workflow automation platform. Our integration allows you to trigger automated actions (email alerts, incident creation, etc.) based on scan results.

🔴 Phishing Awareness

Never click links in unsolicited emails. Verify sender addresses carefully. Look for misspellings and urgent language.

🔵 Password Security

Use unique, strong passwords for every account. Enable multi-factor authentication wherever possible. Use a reputable password manager.

🟢 Safe Browsing

Check URLs before clicking. Look for HTTPS indicators. Avoid downloading files from untrusted sources.

🟣 Social Engineering

Be skeptical of unexpected requests for information or action. Verify identities through separate channels. Don't share sensitive data over unsecured communications.

🟡 AI Safety

Be cautious with AI-generated content. Don't paste sensitive data into public AI tools. Watch for prompt injection attempts in shared documents.

01Keep all software and operating systems updated to the latest versions.
02Use a VPN on public Wi-Fi networks. Avoid accessing sensitive accounts on public networks.
03Regularly review account permissions and revoke access for unused applications.
04Enable automatic screen lock on all devices. Use biometric authentication when available.
05Back up important data regularly using the 3-2-1 rule (3 copies, 2 media types, 1 offsite).
06Verify before trusting. Scan suspicious items with CyberShield AI before engaging.

⚠️ Not a Replacement for Human Judgment

AI analysis should supplement, not replace, human security expertise. Always verify critical findings with qualified professionals.

⚠️ False Positives & Negatives

AI can produce incorrect assessments. A "Safe" verdict doesn't guarantee safety, and a "High Risk" verdict may be a false alarm.

⚠️ Training Data Limitations

LLMs are trained on historical data and may not recognize novel attack vectors that emerged after their training cutoff.

⚠️ Context Dependency

Accuracy improves with better context. Scans without context information may produce less reliable results.
