======================== ZeroTrust One Updates (v2)
========================

-   Rename platform to: ZeroTrust One.
-   Chrome Extension is a primary component, not an optional add-on.
-   Add a top-right “Download Extension” button in the desktop/web UI.
-   Use a Desktop Application (PySide6) + Chrome Extension + FastAPI
    backend architecture.
-   AI stack:
    -   Fine-tuned Transformer (primary detector)
    -   Groq (reasoning & explanations)
    -   DeepSeek (cross-validation)
-   Implement a Unified AI Decision Engine. Do NOT expose per-model
    confidence scores or internal voting. Show only: • Threat Level •
    Overall Confidence • Risk Score • Final Recommendation
-   Produce a natural 5–6 line explanation for every result.
-   Follow Zero Trust + Multi-Layer Defense for every scan.
-   Never fabricate alerts, analytics, logs, or live threat data.
-   Include a built-in Demo Dataset: phishing emails, safe emails, AI
    documents, human documents, fake invoices, genuine invoices,
    malicious URLs, safe URLs, deepfake images, authentic images, scam
    PDFs, genuine PDFs.
-   Desktop dashboard should include: reports, analytics, history,
    settings, AI explanations, extension management.
-   Chrome extension should support: webpage scanning, phishing
    detection, PDF analysis, suspicious forms, downloads, right-click
    analysis, live warnings, browser timeline, AI sidebar.
-   UI inspiration: Huly, Linear, Arc Browser, Notion. Avoid generic
    AI-generated layouts.
-   Enterprise-first experience: Desktop App + Chrome Extension + AI
    Backend + Reporting.


========================
ZeroTrust One Updates (v3)
========================

Authentication & User Management
- Enterprise Sign Up / Sign In
- Google OAuth Login
- Forgot Password
- Remember Me
- Secure Logout
- User Profile & Settings
- JWT Authentication
- Password Hashing (bcrypt)
- Refresh Tokens
- CSRF Protection
- XSS Protection
- SQL Injection Protection
- Role-Based Access Control (RBAC)
- Device Session Tracking

Database Integration
- PostgreSQL
- SQLAlchemy ORM
- Alembic Migrations

API Key & Secrets Management
- Support 4–5 AI provider API keys.
- Never hardcode API keys.
- Store all keys securely in Secrets (.env during development, secure secrets in production).
- Admin can add, update, remove, enable, or disable API keys.
- Initially support: Groq, DeepSeek, Google Gemini, OpenAI, and one custom provider.
- Backend retrieves keys securely.
- API keys are never exposed to the frontend, extension, browser, logs, or reports.
- Gracefully handle invalid or expired keys.

Enterprise Architecture
Desktop App (PySide6) + Chrome Extension + FastAPI + PostgreSQL + Unified AI Engine.


Review & Rating System

- Users can rate the overall scan experience using a 1–5 star rating.
- Users can submit written reviews and feedback after each scan.
- Reviews may optionally include categories such as:
  - Detection Accuracy
  - User Interface
  - Scan Speed
  - AI Explanation Quality
  - Overall Experience
- Users can edit or delete their own reviews.
- The dashboard shall display the average application rating.
- The administrator shall have a moderation panel to review, approve, hide, or remove inappropriate reviews.
- Reviews and ratings shall be stored securely in the PostgreSQL database.
- User feedback can be used to improve future AI models and the user experience.
